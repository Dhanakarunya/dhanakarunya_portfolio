Understanding The Sensor

Goal


To integrate the DHT11 sensor with the ESP32 microcontroller and log temperature and humidity data to Google Sheets via a Web App.


DHT11 Sensor Overview


The DHT11 is a basic, low-cost digital temperature and humidity sensor. It uses a capacitive humidity sensor and a thermistor to measure ambient air conditions and outputs the data via a digital signal. While it's less accurate than the DHT22, it is suitable for many simple IoT applications.

# Feature DHT11
1 Temperature Range 0 to 50 °C
2 Temperature Accuracy ±2 °C
3 Humidity Range 20% to 90% RH
4 Humidity Accuracy ±5% RH
5 Sampling Rate 1 Hz (1 reading/sec)
6 Resolution Low
7 Data Format Digital (single wire)
8 Durability Less durable
9 Price Cheaper

Microcontroller - ESP32

The ESP32 is a powerful, low-cost microcontroller with built-in Wi-Fi and Bluetooth capabilities. It is ideal for IoT projects that require internet connectivity, such as uploading sensor data to Google Sheets.

# Feature Details
1 Microcontroller Tensilica Xtensa LX6 Dual Core
2 Clock Speed Up to 240 MHz
3 Flash Memory Typically 4MB or more
4 RAM ~520 KB SRAM
5 Wi-Fi 802.11 b/g/n
6 Bluetooth v4.2 BR/EDR and BLE
7 GPIO Pins 34 (varies by board)
8 Operating Voltage 3.0V to 3.6V (commonly 3.3V)
9 Power Modes Active, Light Sleep, Deep Sleep
10 Interfaces UART, SPI, I2C, PWM, ADC, DAC, etc.

Common Errors and Fixes


1.HTTP Response Code:

The request didn’t reach the server.

Common Causes:
  • Using http:// instead of https:// (Google Scripts require HTTPS)
  • Web App not deployed properly (must be "Web App" and accessible to "Anyone")
  • No internet or DNS failure
  • Using WiFiClient instead of WiFiClientSecure
  • Incorrect Web App URL or typos
Fix
  • Use HTTPS in URL
  • Deploy script as Web App with access set to "Anyone"
  • Use WiFiClientSecure client; client.setInsecure();

2.HTTP Response Code: -11

The ESP timed out waiting for a server response.

Common Causes:
  • Slow or no response from Google server
  • Wrong URL or incorrect deployment
  • Using HTTP instead of HTTPS
  • Weak Wi-Fi signal
Fix
  • Ensure URL is HTTPS and correct
  • Use WiFiClientSecure
  • Check deployment access
  • Increase timeout if needed

3.Fatal estpool Error

Usually due to toolchain, serial communication issues, or corrupt library installation.

Fix
  • Reinstall board package or libraries
  • Download from wch-ic.com
